# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| main / latest production deployment | :white_check_mark: |
| older tagged releases | :x: |

Only the version currently deployed to production is actively supported with security fixes.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately by emailing **cohenilan@gmail.com**.

Do **not** open a public GitHub issue for security vulnerabilities — this could expose the issue to malicious actors before a fix is available.

Please include:
- A description of the vulnerability and its potential impact
- Steps to reproduce
- Any relevant logs, screenshots, or proof-of-concept code

## Response Time Commitment

We aim to:
- Acknowledge receipt of your report within **3 business days**
- Provide an initial assessment within **7 business days**
- Release a fix or mitigation within **30 days**, depending on severity

## Out of Scope

The following are considered out of scope for vulnerability reports:
- Issues in third-party dependencies that are already publicly disclosed and unpatched upstream
- Social engineering or phishing attacks against staff
- Denial-of-service attacks against shared infrastructure
- Missing security headers on non-sensitive static assets
- Vulnerabilities requiring physical access to a user's device
